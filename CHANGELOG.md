# Changelog

## 2026-04-06

### Summary

本次更新主要修复了短信实时展示重复、固件网络状态误判、以及串口与 URC 并发导致的若干稳定性问题，整体提升了短信收发、消息中心实时更新和设备状态上报的可靠性。

### Firmware

- 优化短信发送判定逻辑，按 `+CMGS`、`OK`、`+CMS ERROR`、`+CME ERROR` 进行更准确判断。
- 增加短信发送参考号 `ref` 返回，便于前端与日志追踪。
- 优化短信接收链路，支持长短信乱序分段拼接并稳定转发。
- 修复接收短信过程中因临时查询本机号导致的串口竞争问题。
- 优化 `query network`、`siminfo` 等查询逻辑，修复 `注册=未知` 与 `IMSI=未知` 的异常情况。
- 修复心跳 `network` 字段缺失、注册状态误判、运营商解析异常等问题。
- 将 MQTT 心跳、响应、原始短信、离线补发统一为可靠发布逻辑。
- 统一串口日志前缀与内容风格，提升运行期排障效率。
- 改进启动阶段与运行阶段的 URC 处理，降低同步查询与异步上报冲突风险。

### Backend

- 接收短信入库时为设备号码字段增加兜底逻辑，避免固件短时上报 `未知` 时影响展示。
- 修复发送记录成功状态仍显示失败原因的问题。
- 保持发送与接收短信广播与数据库记录的一致性。

### Frontend

- 修复实时消息中心和发送记录在 WebSocket 状态更新时重复显示同一短信的问题。
- 实时更新改为按短信 `id` 更新或插入，状态变化时原地刷新，不再重复追加或跳位。
- 优化短信发送成功提示与参考号展示。
- 优化接收短信收件人号码兜底显示，避免出现 `未知` 优先展示。
- 修复按手机号筛选时发送与接收短信显示不全、总数统计和分页异常的问题。

### Known Behavior

- 某些模组环境下 `CID=8` 可能持续处于激活状态，当前版本已改为记录并继续启动，不再采用会影响注册状态的强制关闭策略。
- 当前主链路不受该行为影响，设备注册、心跳、短信发送与接收均可正常工作。

### Commits

- `04b1fcc` `fix duplicate sms updates and harden modem state handling`
- `95c1799` `fix network query fallback and siminfo parsing`

## 2026-04-12

### Summary

本次更新主要围绕 Web 后台消息中心与首页布局做收口与可用性增强，并修复 Docker 前端构建因脚本语法错误导致失败的问题。

### Backend

- 为短信列表接口恢复 `phone` 筛选参数，用于按手机号过滤（发送方/接收方）。
- 保留批量删除短信历史接口（物理删除服务端保存记录）。

### Frontend

- 消息中心筛选项改为“全部手机号”，并包含发送方与接收方手机号。
- 消息中心列表移除“推送状态”列，仅在详情中展示推送状态。
- 新增“刷新”按钮：重置筛选为默认（查询全部）并回到第一页，同时清除当前页勾选。
- 修复推送配置勾选后状态文案在页面初始加载未同步的问题（已启用/未启用）。
- 修复手机号筛选选项生成处多余的 `}` 导致前端打包失败的问题。
- 优化 PC/移动端筛选栏布局与首页设备区块左右列宽比例。
