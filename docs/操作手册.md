# SMS Forwarder MQTT 操作手册

本文档面向日常使用 Web 后台的操作人员，内容以当前代码实现为准。

## 1. 登录

访问地址取决于部署方式：

- 直接运行服务端：`http://服务器IP:3000`
- 通过根目录 Compose 部署：`http://服务器IP:34567`

默认账号：

- 用户名：`admin`
- 密码：`admin123`

登录接口使用 HTTP Basic Auth。登录页调用 `/api/login` 做账号校验，通过后其余接口继续使用相同账号密码访问。

## 2. 首页布局

当前后台主要由以下区域组成：

- 顶部消息中心
- 设备概览卡片
- 设备工具箱
- 推送配置区域

设备工具箱当前包含以下标签页：

- 发送短信
- 信息查询
- 网络测试
- 飞行模式
- AT
- 定时任务
- 诊断

## 3. 设备管理

设备列表显示：

- MAC 地址
- 当前号码
- IP 地址
- 信号强度
- 在线状态
- 备注

当前代码支持两项人工维护信息：

1. 设备备注
2. 设备手机号覆盖值

说明：

- 设备在线状态按最近 120 秒是否有心跳判断
- 如果设置了手机号覆盖值，前端优先显示覆盖值而不是固件上报值

## 4. 消息中心

### 4.1 列表能力

消息中心支持：

- 分页查看
- 关键词搜索
- 按手机号筛选
- 按方向筛选
- 按推送状态筛选
- 当前页多选批量删除

其中：

- `direction` 为 `inbound` 或 `outbound`
- 页面上的“状态”筛选对应推送状态：`success`、`failed`、`pending`

### 4.2 消息详情

消息详情当前可查看：

- 所属设备
- 发件人 / 目标号码
- 短信正文
- 时间戳
- 短信方向
- 消息来源
- 错误信息
- 请求 ID
- 是否为补发消息
- 入队时间
- 各推送渠道状态

### 4.3 删除消息

支持两种删除方式：

- 单条删除：`DELETE /api/messages/:id`
- 批量删除：`POST /api/messages/batch-delete`

删除时会同时移除关联的推送历史记录。

## 5. 发送短信

操作路径：

1. 选择在线设备
2. 打开“发送短信”页签
3. 输入目标号码和短信内容
4. 点击“立即发送”

发送后：

- 服务端会先插入一条 `pending` 状态的出站短信记录
- 设备返回结果后再更新为成功或失败
- 发送记录会显示在工具箱右侧的“发送记录”区域

如果设备处于漫游状态，前端会给出费用风险提示。

## 6. 信息查询

“信息查询”页签当前内置以下操作：

- 固件信息，对应 `query: ati`
- 信号质量，对应 `query: signal`
- SIM 卡信息，对应 `query: siminfo`
- 网络状态，对应 `query: network`
- WiFi 状态，对应 `query: wifi`

这些查询都会走 `POST /api/cmd/:mac` 接口。

## 7. 网络测试

“网络测试”页签通过 `ping` 命令触发设备侧测试。

特点：

- 当前设备必须在线
- 超时时间比普通命令更长
- 如果当前设备网络状态无法确认或处于漫游，前端会给出额外提示

## 8. 飞行模式

“飞行模式”页签提供三种操作：

- 开启飞行模式：`status = 1`
- 关闭飞行模式：`status = 0`
- 查询当前状态：`status = -1`

接口仍为 `POST /api/cmd/:mac`，`action` 为 `flight_mode`。

## 9. AT 终端

AT 页签可直接下发 `at` 命令，并提供快捷按钮：

- `ATI`
- `AT+CSQ`
- `AT+CIMI`
- `AT+ICCID`
- `AT+CNUM`
- `AT+COPS?`
- `AT+CEREG?`
- `AT+CGACT?`

适合用于：

- 查看模组信息
- 检查信号
- 检查注册状态
- 临时调试网络问题

## 10. 定时任务

### 10.1 支持的任务类型

当前支持两类任务：

- 定时短信任务
- 定时流量消耗任务

### 10.2 支持的执行方式

- `once`：一次性执行
- `interval`：每 N 天在固定时间执行一次

### 10.3 创建规则

创建任务时：

- 设备必须在线
- 短信任务必须填写目标号码和内容
- 流量任务的 `traffic_kb` 必须在 `1-10240` 之间
- 必须提供发送时间

### 10.4 任务状态

当前前端展示的状态包括：

- `active`：执行中
- `paused`：已暂停
- `completed`：已完成
- `failed`：已失败

### 10.5 任务详情

任务详情中可以查看：

- 任务类型
- 执行方式
- 目标号码或目标流量
- 下次执行时间
- 当前状态
- 历史执行记录

周期任务支持暂停、恢复与删除；一次性任务执行完成或失败后通常不再恢复。

## 11. 推送配置

当前已接入的推送渠道：

- Telegram
- 钉钉
- PushPlus
- Server 酱
- 飞书
- Bark
- Gotify
- POST JSON
- 自定义模板

后台会保存渠道配置，并在收到短信后异步执行推送。推送结果会写入 `push_history`，并回流到消息详情展示。

## 12. 诊断面板

“诊断”页签用于查看设备健康状态，当前依赖固件心跳中的诊断字段。

主要指标：

- `URC 积压`
- `离线短信`
- `可用内存`
- `最低可用内存`
- `PDU 等待状态`
- `延迟 PDU 状态`

同时提供诊断历史接口：

- `GET /api/devices/:mac/diag-history?range=1h`
- `GET /api/devices/:mac/diag-history?range=24h`
- `GET /api/devices/:mac/diag-history?range=7d`

## 13. 常见问题

### 13.1 登录失败

- 检查账号密码是否正确
- 确认是否已经修改过密码
- 确认请求头里带了正确的 Basic Auth

### 13.2 设备显示离线

- 检查设备是否仍在上电运行
- 检查 WiFi 与 MQTT 配置
- 检查服务端 MQTT Broker 是否可达
- 如果超过 120 秒没有新心跳，界面会自动判定离线

### 13.3 看得到设备但不能发命令

- 当前设备可能已离线
- MQTT 客户端可能未连接
- 设备可能未返回响应，导致请求超时

### 13.4 消息状态一直是“推送中”

这通常表示：

- 该消息没有启用任何推送渠道
- 或已启用渠道，但尚未写入成功 / 失败记录

请重点检查推送配置是否启用、目标地址是否可达。

### 13.5 定时流量任务无法执行

如果服务端检测到设备处于漫游状态，会阻止流量消耗型操作，包括定时流量任务。
