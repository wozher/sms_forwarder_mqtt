### 1.1 访问 Web 后台

```
地址: http://192.168.31.197:34567
默认账号: admin
默认密码: admin123
```

- 下方左侧为设备概览（在线设备列表与状态）

## 2. 登录与认证

### 2.1 登录

1. 打开浏览器访问 `http://192.168.31.197:34567`
2. 输入用户名和密码
3. 点击“登录”按钮，或直接按回车
4. 登录成功后进入管理后台

### 2.2 修改密码

1. 点击右上角用户菜单 `👤 admin ▼`
2. 选择“🔑 修改密码”
3. 输入当前密码与新密码
4. 确认后系统会自动退出，需要用新密码重新登录

**接口：**

```http
POST /api/password
Authorization: Basic {base64(username:oldPassword)}
Content-Type: application/json

{"newPassword": "新密码"}
```

## 3. 设备管理

### 3.1 查看设备列表

主界面显示所有已注册设备，包含：

- MAC 地址
- IP 地址
- 手机号（SIM 卡号码）
- 信号强度
- 在线 / 离线状态

### 3.2 选择设备

- 点击设备卡片可切换当前设备
- 工具箱右上角也可通过下拉框快速切换设备
- 有在线设备时，系统会自动选择一个在线设备作为默认目标

### 3.3 心跳机制

- 设备每 60 秒发送一次心跳
- 超过 2 分钟无心跳，设备会显示为离线

## 4. 短信收发

### 4.1 查看接收短信

- 消息中心支持分页浏览历史短信（每页 10 条）
- 支持搜索、按手机号筛选、按方向筛选、按状态筛选
- 消息列表以表格展示，操作列仅提供“详情”用于查看完整信息
- 推送状态仅在“详情”中展示（列表中不显示）
- 支持当前页勾选后批量删除短信历史记录（物理删除服务端历史记录）

### 4.2 发送短信

1. 在设备工具箱中选择“📤 发送短信”
2. 输入目标号码和短信内容
3. 点击“📨 立即发送”
4. 等待设备返回发送结果

## 5. 定时任务

### 5.1 创建任务

1. 在设备工具箱中选择“⏰ 定时任务”
2. 输入目标号码和短信内容
3. 选择一次性发送或周期发送
4. 设置时间后提交

### 5.2 管理任务

- 查看任务详情和执行历史
- 暂停 / 恢复周期任务
- 删除不再需要的任务
- 同一设备可创建多条定时任务（短信/流量可混合）

### 5.3 定时流量消耗任务

1. 在设备工具箱中选择“⏰ 定时任务”
2. 任务类型选择“定时消耗流量”
3. 设置目标流量（KB）和执行时间/周期
4. 创建后系统按计划执行流量消耗

提示：目标流量建议从小值开始（如 64KB / 128KB），逐步验证。

## 6. 设备工具箱

当前版本包含以下功能：

- 顶部可通过下拉框快速切换目标设备（MAC + 手机号 + 在线状态）

| 功能 | 说明 |
|------|------|
| 推送配置 | 配置消息推送渠道 |
| 发送短信 | 向当前设备发送短信 |
| 信息查询 | 查询设备与模组状态 |
| 网络测试 | 执行 Ping 测试 |
| 飞行模式 | 开启 / 关闭飞行模式 |
| AT终端 | 直接发送 AT 指令 |
| 定时任务 | 创建和管理定时短信任务 |

### 6.1 信息查询

支持以下类型：

- 固件信息
- 信号质量
- SIM 卡信息
- 网络状态
- WiFi 状态

### 6.2 网络测试

- 向 `8.8.8.8` 发起 Ping 测试
- 用于检查模组网络连通性

### 6.3 飞行模式

- 开启飞行模式
- 关闭飞行模式
- 查询当前飞行模式状态

### 6.4 AT 终端

- 可手工发送 AT 指令
- 也提供常用快捷按钮，如 `ATI`、`AT+CSQ`、`AT+CIMI`

## 8. MQTT 命令协议

### 8.1 下发主题

`sms_forwarder/cmd/{MAC}`

### 8.2 响应主题

`sms_forwarder/resp/{MAC}`

### 8.3 支持的 Action

| Action | 参数 | 说明 |
|--------|------|------|
| `send_sms` | `{phone, content}` | 发送短信 |
| `ping` | `{}` | Ping 测试 |
| `query` | `{type}` | 查询设备信息 |
| `consume_traffic` | `{targetKb}` | 按 KB 消耗流量 |
| `query_sim` | `{}` | 查询 SIM 信息 |
| `flight_mode` | `{status}` | 飞行模式控制 |
| `at` | `{cmd}` | 执行 AT 指令 |
| `reset` | `{}` | 重启设备 |

命令可附带 `requestId` 字段，设备响应会原样返回该字段。

## 9. API 接口参考

所有接口默认都需要 HTTP Basic Auth 认证。

### 9.1 认证接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/login` | POST | 登录验证 |
| `/api/password` | POST | 修改密码 |

### 9.2 设备接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/devices` | GET | 设备列表 |
| `/api/cmd/:mac` | POST | 发送命令 |

### 9.3 消息接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/messages` | GET | 短信历史，支持分页与筛选 |
| `/api/stats` | GET | 统计信息 |

`/api/messages` 支持 `search`（关键词）、`phone`（手机号）、`direction=inbound/outbound` 与 `status=success/failed/pending` 过滤。

删除短信历史记录：

- `POST /api/messages/batch-delete` 批量删除（推荐）
- `DELETE /api/messages/:id` 单条删除（目前前端不提供入口）

### 9.4 推送接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/push-config` | GET | 获取推送配置 |
| `/api/push-config` | POST | 保存推送配置 |
| `/api/push-history` | GET | 获取推送历史 |
| `/api/push-history/sms/:smsId` | GET | 查询某条短信的推送记录 |
| `/api/push-history/:id/retry` | POST | 重试推送 |

### 9.5 定时任务接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/schedule` | GET | 定时任务列表 |
| `/api/schedule` | POST | 创建定时任务 |
| `/api/schedule/:id` | GET | 获取任务详情 |
| `/api/schedule/:id` | PUT | 更新任务 |
| `/api/schedule/:id` | DELETE | 删除任务 |

## 10. 故障排查

### 10.1 登录失败

- 检查用户名和密码是否正确
- 修改密码后请重新登录
- 若浏览器保存了旧密码，清除后再试

### 10.2 设备不在线

- 检查 ESP32 是否上电
- 检查 WiFi 和 MQTT 配置是否正确
- 检查设备是否能连接到 MQTT Broker

### 10.3 短信发送失败

- 检查 SIM 卡状态和信号
- 在“信息查询”或“AT 终端”里确认模组工作正常
- 失败的手动发送与定时发送都会保存在发送记录中，可在“设备工具箱 -> 发送短信”查看

### 10.5 漫游状态下操作规则

- 漫游允许：手动 Ping、手动短信、定时短信、定时流量消耗任务
- 漫游禁止：临时高流量消耗操作
- 定时流量为人工配置任务，默认视为已评估流量成本

### 10.4 Docker 部署数据保留

- 请保留 `./data:/app/data` 挂载
- 只要宿主机 `data` 目录不被删除，重启或重建容器不会丢失短信记录、用户和推送配置
